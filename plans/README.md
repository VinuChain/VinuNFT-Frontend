## Dependency Notes

- Plan 002 should land before any push to widen public image minting, because
  image uploads are currently intentionally gated by allowlist and rate limits.
- Plan 003 is independent, but its review should reuse the same security posture
  as Plan 002: user-controlled content must stay bounded, escaped, and
  allowlisted.
- Plan 004 can be done independently for markdown-only. If enabling HTML, review
  it with the same content-safety rigor as Plan 003.
- Plan 001 and Plan 005 both risk unbounded RPC scans. If either discovers that
  client-only data fetching is not viable, stop and plan an indexer instead.
- Plan 006 is now a broad bridge-port plan. It can run independently, but it
  touches API functions, wallet transaction handling, header navigation, and
  marketplace payment-token UI.

## Repo Recon Summary

- Framework: Gatsby 5 / React 18.
- Package manager: Yarn 1 (`packageManager`: `yarn@1.22.22`).
- Node: package engines require `>=18 <23`; README recommends Node 20.
- Verification gates:
  - `yarn lint`
  - `yarn test`
  - `yarn audit:triage`
  - `yarn build`
- Test shape: Node `node:test` source-level audit regressions in
  `test/audit-regressions.test.js`; no default live RPC/browser tests.
- Deployment shape: `yarn build` produces Gatsby `public/` plus serverless
  Gatsby Function support under `src/api/`.
- Style conventions: 4-space Prettier, function components, React hooks,
  Recoil for shared app state, ethers contracts using `src/config.js` and
  `src/common/abi.js`.
- Current planned-at commit for all plans: `a00888a`.
- Current worktree at planning time had broad pre-existing modifications; each
  executor must run the drift check in its plan before editing.
